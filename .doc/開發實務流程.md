# 🚀 Docker 雙棲開發實務流程

這份文件記錄了本專案的標準開發工作流 (Workflow)。
無論是換一台新電腦，或是新同事加入，只要照著以下步驟，就能在 **5 分鐘內** 建立起完美的本地熱重載開發環境！

---

## 🛠 階段一：環境初始化 (Initialization)

當你第一次 Clone 這個專案時：

1. **下載專案代碼**
   ```bash
   git clone https://github.com/liwenchiou/dockerDevDEMO.git
   cd dockerDevDEMO
   ```

2. **建立本機環境變數 (`.env`)**
   因為資安考量，`.env` 不會上傳到 GitHub。我們需要從範本複製一份：
   ```bash
   cp .env.example .env
   ```
   *(可選)* 打開 `.env` 修改你的資料庫連線密碼 (`DB_PASSWORD`)。

3. **建立本地開發覆寫設定 (`override.yml`)**
   因為 `docker-compose.yml` 是正式上線版，我們需要開發版專屬的掛載與熱重載設定。此檔案同樣被 `.gitignore` 忽略：
   ```bash
   cp docker-compose.override.example.yml docker-compose.override.yml
   ```

---

## 💻 階段二：日常開發 (Daily Development)

每天上班開始寫 code 的流程：

1. **一鍵啟動所有環境**
   ```bash
   docker compose up -d --build
   ```
   *(這會自動合併 `docker-compose.yml` 與 `docker-compose.override.yml`，並啟動 Node.js 與 Postgres)*

2. **享受熱重載開發 (Hot Reload)**
   - 伺服器會自動執行 `nodemon` 與 `tailwindcss watch`。
   - 你可以直接在 VSCode 內修改 `routes/`、`views/` 等程式碼。
   - 存檔後，前往 `http://localhost:3000` **重新整理頁面**，即可看到最新修改。

3. **開發完畢，關閉環境 (下班)**
   ```bash
   docker compose stop
   ```
   *(注意：使用 `stop` 會保留資料庫紀錄。如果想要連資料庫都砍掉重練，請改用 `docker compose down -v`)*

---

## 📦 階段三：版本控制與發布 (Git & Deploy)

1. **提交程式碼**
   完成功能後，執行標準的 Git 流程：
   ```bash
   git add .
   git commit -m "feat: 完成了超酷的新功能"
   git push
   ```

2. **雲端部署機制**
   - 程式碼 Push 到 GitHub 後，因為沒有 `.env` 和 `override.yml`，雲端伺服器只會讀取到乾淨且安全的 `docker-compose.yml`。
   - 雲端環境將自動使用 Docker Volume (保險箱) 來保護實體資料，且不會啟動 `nodemon`，確保效能與安全性最佳化。

🎉 **打完收工，準備下班！**
